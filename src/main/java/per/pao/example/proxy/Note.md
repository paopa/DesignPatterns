# Proxy Pattern

## Note
>
> 又稱：代理模式
> 
> 範例為 JOJO 的奇妙冒險
>
> 優點： 低耦合、易擴充、高靈活
> 缺點： 間接訪問可能請求延遲、增加工作量
>
> 主要用途：
>   1. 遠端代理
>   2. 虛代理
>   3. 保護代理
>   4. 智慧指引代理
>   5. 寫時複製代理（Copy-on-Write）
>   6. 快取代理
>   7. 防火牆代理
>   8. 同步化代理
